1. Security & safety first

Never share API keys or worlo Teams invite codes in public places — including Discord servers, Slack channels, GitHub issues, screenshots, screen recordings, or client-side code. Anyone with your key or code can use your agent and any connected services it can access.

Agents on worlo can read email, send messages, browse websites, and act on connected apps depending on how you configure them. Treat every agent like a privileged account:

  • Share carefully. Only give API keys and invite codes to people you trust inside your organization.
  • Use environment variables. Store keys in server-side secrets (Vercel, Railway, AWS Secrets Manager, etc.) — never hard-code them in frontend JavaScript.
  • Revoke immediately if a key or code is exposed. Revoke from the dashboard under API Keys or worlo Teams.
  • Limit permissions. Turn off send/edit permissions unless the agent truly needs them. Restrict allowed websites when possible.
  • Review connected apps. Disconnect OAuth integrations you no longer use.

2. Connecting services

worlo agents can use OAuth-linked apps (Slack, Google Calendar, Gmail, GitHub, Notion, Discord, Microsoft Teams, and more) when you connect them in the dashboard.

Step-by-step

  1. Sign in to the dashboard and open Connections.
  2. Find the service you want and click Connect.
  3. Complete the OAuth sign-in in the popup or browser window.
  4. Open your agent, click Edit, and enable the connections that agent should use.
  5. Save the agent so its token picks up the linked apps.

Connections are per-account. Each agent chooses which linked apps it may use — an agent without Gmail connected cannot read your email even if your account has Gmail linked elsewhere.

3. Using API keys

Each agent can have one API key (nra_ prefix). The key turns that agent into an HTTP endpoint with its prompt, model, permissions, and linked connections.

Creating a key

  1. Save your agent with the settings and connections you want.
  2. Open API Keys in the dashboard (or click API Key on an agent card).
  3. Select the agent and click Generate API Key.
  4. Copy the key immediately from the popup — you will not be able to view it again after closing the dialog.

If you lose a key, revoke it and generate a new one. The old key stops working instantly.

Calling your agent

curl -X POST 'https://YOUR_PROJECT.supabase.co/functions/v1/agent-invoke' \
  -H 'Content-Type: application/json' \
  -H 'X-Agent-Key: nra_your_key_here' \
  -d '{"message":"Hello from my app"}'

Send a JSON body with message (required) and optional history for multi-turn conversations.

4. Writing good backend prompts

When you use OpenAI with a custom API key, worlo can wrap your instructions in the worlo backend prompt — a platform layer that teaches the agent how to use connected apps, browser tools, and permission boundaries safely.

Best practices

  • Be specific about the role. “You are our customer support agent for Acme Corp” works better than “You are helpful.”
  • Define boundaries. Say what the agent should and should not do (e.g. “Never send emails without explicit user confirmation”).
  • List allowed websites when unrestricted browsing is off — the agent will only browse those URLs.
  • Keep instructions concise. Long prompts cost more tokens and can dilute focus.
  • Leave worlo backend prompt enabled unless you have a reason to send raw instructions only — it adds safe defaults for tool use and connected apps.

Example agent instruction

You are the ops assistant for our support team.
- Answer questions using Slack history and Google Calendar when relevant.
- Draft replies but do not send messages unless the user says "send it".
- Be concise and cite sources when pulling from connected apps.

5. Connecting worlo API keys to projects

Use worlo as a backend agent from your own app, script, or automation — always from server-side code.

Node.js example

const WORLO_AGENT_KEY = process.env.WORLO_AGENT_KEY;

async function askAgent(message) {
  const res = await fetch(
    'https://YOUR_PROJECT.supabase.co/functions/v1/agent-invoke',
    {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'X-Agent-Key': WORLO_AGENT_KEY,
      },
      body: JSON.stringify({ message }),
    }
  );
  const data = await res.json();
  if (!res.ok) throw new Error(data.error || 'Agent request failed');
  return data.reply;
}

Python example

import os, requests

WORLO_AGENT_KEY = os.environ["WORLO_AGENT_KEY"]
URL = "https://YOUR_PROJECT.supabase.co/functions/v1/agent-invoke"

def ask_agent(message: str) -> str:
    r = requests.post(
        URL,
        headers={"Content-Type": "application/json", "X-Agent-Key": WORLO_AGENT_KEY},
        json={"message": message},
        timeout=120,
    )
    r.raise_for_status()
    return r.json()["reply"]

Replace YOUR_PROJECT with your Supabase project URL from the dashboard API docs section. Never expose the key in browser code, mobile apps, or public repos.

6. worlo Teams invite codes

worlo Teams is a separate desktop and browser app for chatting with your agents. Generate invite codes from the dashboard to let teammates join.

  1. Open worlo Teams in the dashboard.
  2. Enable Teams access for the agent you want to share.
  3. Generate an invite code and copy it immediately.
  4. Share the code privately with teammates — not in public channels.

Invite codes grant access to the linked agent. Revoke codes you no longer need and disable Teams on agents when sharing should stop.
